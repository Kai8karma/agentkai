01

The demo is on Vercel. Your data never will be.

Every other CRM audit tool works the same way: you hand over API access, your contacts and deals flow into their cloud, and a few days later you get a slide deck scored on a curve nobody can check. That's not a design choice they can undo — hosted SaaS is the product, so "your data stays local" was never on their roadmap.

gtmos runs the other way. pip install -e . puts the audit engine, the funnel model, and the local dashboard on your machine, reading your HubSpot data through a token you hold (GTMOS_HUBSPOT_TOKEN). The scoring code never calls a network API — it's pure Python, reproducible from the same input every time. The only two systems gtmos ever talks to outside your machine are HubSpot (your token) and the claude CLI (your own Claude Code auth) — both documented, both yours.

This page is hosted. The website is allowed to leave your machine. Your contact list, your deal stages, your customer emails — those don't, because there's no server in the path for them to leave through.

05

What the $-number actually means

The headline is a dollar figure, and it's built from two deterministic models — no LLM ever touches it.

The audit side flags records as unreachable when they have no valid email and no valid phone (validity or freshness below threshold), then converts the unreachable share into dollars using your ACV and a baseline conversion rate: unreachable_pct × total_records × ACV × 2%.

The funnel side looks at deal stages instead of contacts: it adds up closed-lost value plus half of stalled open-deal value (closed-lost + 0.5 × stalled), imputing a median value for any deal missing a dollar amount so incomplete data doesn't zero out the estimate.

Honest caveat: this is an order-of-magnitude estimate, not a forecast. The 2% baseline conversion rate and the 0.5× stalled-deal discount are stated assumptions, not measured facts about your pipeline — both are editable inputs (--acv, --stall-days), and the methodology is published so you can check or reimplement it yourself.

06

Security

  • One chokepoint for every LLM call. run_harness() in gtmos/harness.py is the only function in the codebase that shells out to claude — enforced by a static scan that fails the build if any other file calls subprocess.*.
  • A receipt for every AI call. Timestamp, command, prompt size, duration, and exit code get written to ~/.gtmos/receipts.jsonl on every harness invocation — agent usage is auditable, not a black box.
  • Your token is never logged. GTMOS_HUBSPOT_TOKEN is read from the environment at call time and held in process memory only — it is never written to gtmos-team.json, report.json, the receipts log, or any other artifact gtmos produces.
  • The dashboard binds to localhost. gtmos serve defaults to 127.0.0.1; binding 0.0.0.0 is refused unless you pass --lan explicitly, and the dashboard itself is read-only.
  • An eval harness gates the claims. gtmos eval runs an audit-determinism check, a receipts-integrity check, and the harness-contract scan before you trust a number — exit 1 on any failure, wired into pre-push.

FAQ

Is my CRM data sent anywhere?

No. The audit and funnel engines run as plain Python on your machine and never make a network call. The only two systems gtmos talks to outside your machine are the HubSpot API (your own token) and the local claude CLI subprocess (your own Claude Code auth) — both are things you already hold, not a new vendor.

What does the AI actually do?

It never touches a score. LLM calls are limited to a handful of commands — prospect, stalk, draft, brief, and play run --draft — that draft outreach copy or research, all routed through one chokepoint (run_harness()) with a receipt written for every call. Audit, funnel, eval, weekly, and play planning are deterministic Python with no model in the loop.

HubSpot only?

Today, yes — the audit and funnel engines are built against HubSpot's contact/company and deal model. The scoring methodology itself is CRM-agnostic; other CRM adapters are on the roadmap, gated on a paying or piloting customer whose primary CRM isn't HubSpot.

How is the leak number computed?

Two published, deterministic formulas: on the audit side, unreachable_pct × total_records × ACV × 2%; on the funnel side, closed-lost value + 0.5 × stalled open value with median imputation for unvalued deals. Full methodology is documented, not a black box — you can check the math or reimplement it yourself.
